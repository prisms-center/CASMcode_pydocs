Enumerate orderings in Si-Ge#

Specify the “prim”#

This step:

  • Defines the primitive crystal structure and allowed degrees of freedom (DoF) for a diamond cubic Si-Ge alloy.

Notes

A primitive crystal structure and allowed degrees of freedom (the “prim”) specifies:

  • primitive cell lattice vectors

  • crystal basis sites

  • global DoF (none in this case)

  • site DoF (Si or Ge occupation in this case).

Here the prim is specified using a Python dict / JSON. The dict / JSON representation can be used to construct one of two CASM classes:

  • xtal.Prim: This is just the prim definition

  • casmconfig.Prim: This is the xtal.Prim, along with the symmetry representations needed for applying symmetry operations to CASM Configuration

prim_data = {
    "title": "SiGe_occ",
    "lattice_vectors": [
        [0.000000000000, 2.800000000000, 2.800000000000],  # 1st lattice vector
        [2.800000000000, 0.000000000000, 2.800000000000],  # 2nd lattice vector
        [2.800000000000, 2.800000000000, 0.000000000000],  # 3rd lattice vector
    ],
    "coordinate_mode": "Fractional",
    "basis": [
        {
            "coordinate": [0.0, 0.0, 0.0],
            "occupants": ["Si", "Ge"],
        },
        {
            "coordinate": [0.25, 0.25, 0.25],
            "occupants": ["Si", "Ge"],
        },
    ],
}

Create a CASM project#

This step:

  • Initializes a diamond cubic Si-Ge alloy CASM project using Project.init.

  • Prints the prim’s factor group symmetry operations using SymCommand.print_factor_group.

Important

Checking the factor group helps to catch errors in the prim. The factor group is used to determine which configurations and basis functions are symmetrically equivalent.

Notes

A CASM project is a directory containing data related to a particular prim. The CASM project directory structure standardizes the location of various files used by multiple CASM methods. This makes it easier to perform the most common operations and easier to share a project with others.

The Project class helps read and write and use CASM project data. A Project is initialized by specifying a prim and project directory location using Project.init. This will:

  1. Check if the prim has a primitive unit cell with a CASM standard lattice orientation

  2. Perform a symmetry analysis

  3. Generate some default directories, data, and settings

  4. Perform a configuration check

If a project already exists at the specified location, the prim argument to Project.init is ignored and the existing project is loaded.

Additional notes:

  • Project files that the user should not typically modify directly, including a copy of the prim, are stored in a hidden .casm sub-directory of the CASM project directory. The presense or absence of the .casm directory is used by CASM to detect a CASM project.

  • A description of symmetry analysis in CASM can be found in the libcasm-xtal usage documentation.

  • SymCommand.print_factor_group prints symmetry operations using notation which roughly follows the conventions of the International Tables for Crystallography.

from casm.project import Project
import pathlib

# Initialize the project
project = Project.init(
    path="SiGe_occ",
    prim=prim_data,
)

# Coordinates can be printed in 'cart' or 'frac' coordinate mode
print("\nFactor group operations:")
project.sym.print_factor_group(coord="frac")

Enumerate symmetrically unique configurations#

This step:

  • Creates an enumeration with id “main” using EnumCommand.get().

  • Enumerates all symmetrically unique Configuration in supercells up to volume 4 using EnumData.occ_by_supercell().

  • Stores results in a ConfigurationSet and writes it to a file (config_set.json) in the enumeration directory.

Notes

A Configuration is a compact representation of the unit cell for a crystal structure that is consistent with the DoF specified in the prim. For this Si-Ge project, a configuration can be specified by:

  • the supercell of the prim that is the unit cell for the crystal structure, and

  • an integer occupation array (i.e. [1, 0, 1, …]), indicating if Si or Ge is on each site in the supercell.

Enumeration directories are located in a CASM project at:

    enum_dir = <project>/enumerations/enum.<enum_id>/

The EnumData class helps interact with the saved Configuration. An EnumData object (enum) is created from the Project using EnumCommand.get() with the enumeration id:

    enum = project.enum.get(id="main")

The EnumData.occ_by_supercell() method:

  • enumerates configurations in a range of supercells specified by their volume with respect to the primitive unit cell volume,

  • inserts enumerated configurations in EnumData.configuration_set, and

  • writes EnumData.configuration_set to the config_set.json file in the enumeration directory.

CASM implements a strict ordering on configurations, and the configuration which compares greatest to all the other symmetrically equivalent configurations in a supercell is said to be in canonical form. By default, only primitive configurations that are in canonical form are stored by EnumData.occ_by_supercell(). Using this default, if configurations are enumerated from the volume 1 supercell on up, then only symmetrically unique configurations are stored.

# Enumerate configurations in supercells with volume 1 to 4
# - Configurations are stored in the enumeration's ConfigurationSet
enum = project.enum.get(id="main")
enum.occ_by_supercell(max=4, min=1)

Iterate over configurations in a ConfigurationSet#

This step:

  • Iterates over the first 20 ConfigurationRecord in EnumData.configuration_set.

  • Prints the configuration’s name, occupation indices, and corresponding atom types.

Notes

For configuration stored in a ConfigurationSet, the configuration name consists of a supercell name and a configuration index. The supercell name (i.e. “SCEL2_2_1_1_0_1_1”) is generated from the transformation matrix that gives the supercell lattice vectors as a linear combination of the primitive cell lattice vectors. The configuration index is a sequential index generated by the ConfigurationSet for configurations in the same supercell.

When iterating over ConfigurationRecord in a EnumData.configuration_set, the ordering is based on sorting the configuration by supercell and then their DoF values, not their configuration name.

# Iterate over the first 20 ConfigurationRecord in the ConfigurationSet,
# and print:
# - the configuration name
# - the occupation indices
# - the corresponding atom types
print(f'{"ConfigurationName":24}{"OccupationIndices":20}{"AtomType":40}')
print("-" * 84)
for i, record in enumerate(enum.configuration_set):
    name = record.configuration_name
    occupation = record.configuration.occupation
    structure = record.configuration.to_structure()
    atom_type = structure.atom_type()
    print(f"{name:24}{str(occupation):20}{str(atom_type):40}")
    if i == 20:
        break

Set parametric composition axes#

This step:

  • Prints the standard choices for independent parametric composition axes using Project.chemical_composition_axes.

  • Selects one choice to use with CompositionAxes.set_current_axes()

  • Prints the selected axes along with formulas for converting between (i) number composition and parametric composition and (ii) chemical potential and parametric chemical potential.

  • Saves the chosen parametric composition axes with CompositionAxes.commit().

Notes

In a crystal with a fixed number of sites, the number of species occupying the same sublattice are not indepedent. In general, a crystal occupied by \(s\) component species will have \(k<s\) independent compositions. As described in [1], CASM converts between compositions expressed as number of species per unit cell and compositions in terms of independent “parametric composition axes” using:

(1)#\[\begin{align} \vec{n} &= \vec{n}_0 + \mathbf{Q} \vec{x} \\ \vec{x} &= \mathbf{R}^{\mathsf{T}} (\vec{n} - \vec{n}_0) \end{align}\]

where:

  • \(\vec{n}\): Vector of shape=(\(s\),), the number of each component species per unit cell (mol_composition).

  • \(\vec{x}\): Vector of shape=(\(k\),), The composition along each composition axis when referenced to the origin composition (param_composition).

  • \(\vec{n}_0\): Vector of shape=(\(s\),), The origin in composition space, as the number of each component species per unit cell.

  • \(\mathbf{Q}\): Matrix of shape=(\(s\), \(k\)), with columns representing the change in composition per unit cell going one unit distance along each independent composition axis.

  • \(\mathbf{R}\): Matrix of shape=(\(s\), \(k\)), such that \(\mathbf{R}^{\mathsf{T}}\mathbf{Q} = \mathbf{Q}^{\mathsf{T}}\mathbf{R} = \mathbf{I}\).

The parametric composition axes are the columns of \(\mathbf{Q}\), \(\vec{q}_i\). Due to preservation of the number of sites per unit cell, \(\sum_{i} Q_{ij} = 0\). If vacancies are allowed, they are included as a component species.

[1]: Computational Materials Science 260 (2025) 114091. DOI: 10.1016/j.commatsci.2025.114091.

Iterate over configurations using ConfigSelection#

This step:

  • Constructs a ConfigSelection using EnumData.config_selection().

  • Uses the ConfigSelection to iterate over ConfigSelectionRecord for configurations stored in the enumeration.

  • Prints the number composition per unit cell for each configuration.

  • Prints the parametric composition for each configuration.

  • Demonstrates getting the order of components in the composition per unit cell array from ConfigSelection.chemical_components.

Notes

ConfigSelection is a helper that can be used to:

  • set certain configurations in an enumeration as “selected” and others as “unselected”,

  • iterate over only selected configurations (by default), or all configurations, or only unselected configurations,

  • evaluate configuration properties, such as the parametric composition or cluster expansion basis function correlations, using data stored in the project, and

  • perform actions on selected configurations, like setup calculation input files or collect calculation results.

ConfigSelection are given a name and stored in a file named config_selection.<name>.json in the enumeration directory when the ConfigSelection.commit() method is used. By default, when a ConfigSelection does not already exist it is constructed with all configurations selected.

When ConfigSelection is constructed or when a certain property of a configuration is requested from a ConfigSelectionRecord, project data like the chemical composition axes or a CASM clexulator is read and used to evaluate the requested property.

Additional notes:

  • Enumerations also allow storing configurations in a list, EnumData.configuration_list, which is stored in a file named config_list.json, to enable storing configurations that are not in the canonical supercell (for example, configurations with a particular supercell orientation).

  • Iterating over a ConfigSelection yields ConfigSelectionRecord for the configurations stored in both EnumData.configuration_set and EnumData.configuration_list.

# Construct a ConfigSelection (by default all configurations are selected)
sel = enum.config_selection(name="main")

# Get the order of components in chemical_comp_per_unitcell:
legend = ", ".join(sel.chemical_components)

# Iterate over the first 20 ConfigSelectionRecord and print:
# - the configuration name
# - the atom types
# - the composition per unit cell
# - the parametric composition

print(
    f'{"ConfigurationName":24}{"AtomType":40}{f"CompPerUnitcell [{legend}]":30}{"ParamComposition":20}'
)
print("-" * 120)
for i, record in enumerate(sel):
    name = record.name
    structure = record.structure
    atom_type = structure.atom_type()
    comp_per_unitcell = record.chemical_comp_per_unitcell
    param_comp = record.chemical_param_comp
    print(
        f"{name:24}{str(atom_type):40}{str(comp_per_unitcell):30}{str(param_comp):20}"
    )
    if i == 20:
        break

Write structure files#

This step:

  • Iterates over selected configurations using ConfigSelection.

  • In the calculation directory for each configuration, writes:

    • the CASM Configuration to a file named config.json,

    • a CASM Structure representation to a file named structure.json, and

    • a VASP POSCAR to a file named structure.vasp.

Notes

A CASM Structure:

  • represents a crystal structure with a 3d lattice,

  • is not restricted to the DoF values allowed by a prim,

  • has built in methods for conversions to and from VASP POSCAR format.

The ConfigSelectionRecord.structure property does the conversion from Configuration to Structure, leaving out any vacancies, using the Configuration.to_structure method. It is also possible get the Structure representation including vacancies using the ConfigSelectionRecord.structure_with_vacancies property.

Files are written to the CASM standard calculation directory for the configuration, which is:

calc_dir = <enum_dir>/training_data/calctype.<calctype_id>/<configname>/

where the default calctype_id is “default”.

from casm.tools.shared.json_io import safe_dump
from casm.tools.shared.text_io import safe_write

for record in sel:
    calc_dir = record.calc_dir
    name = record.name
    safe_dump(record.configuration.to_dict(), calc_dir / "config.json", quiet=True)
    safe_dump(record.structure.to_dict(), calc_dir / "structure.json", quiet=True)
    safe_write(
        record.structure.to_poscar_str(title=name),
        calc_dir / "structure.vasp",
        quiet=False,
    )
