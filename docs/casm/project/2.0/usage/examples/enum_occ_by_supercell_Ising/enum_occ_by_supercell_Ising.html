Enumerate orderings in the 2D Ising model#

Specify the “prim”#

This step:

  • Defines the primitive crystal structure and allowed degrees of freedom (DoF) for a 2-dimensional Ising model system.

  • Uses occupants named “A.down” and “A.up” to represent “A” atoms with collinear magnetic spin values of -1 and +1, respectively.

Notes

A primitive crystal structure and allowed degrees of freedom (the “prim”) specifies:

  • primitive cell lattice vectors

  • crystal basis sites

  • global DoF (none in this case)

  • site DoF (A.up or A.down occupation in this case).

Here the prim is constructed using the xtal.Prim.

  • xtal.Prim: This is just the prim definition

  • casmconfig.Prim: This is the xtal.Prim, along with the symmetry representations needed for applying symmetry operations to CASM Configuration

import libcasm.xtal as xtal
import numpy as np

# Lattice vectors:
lattice = xtal.Lattice(
    np.array(
        [
            [1.0, 0.0, 0.0],  # 1st lattice vector
            [0.0, 1.0, 0.0],  # 2nd lattice vector
            [0.0, 0.0, 10.0],  # 3rd lattice vector (will have no effect)
        ]
    ).transpose()  # <--- note transpose
)

# Basis sites positions, as columns of a matrix,
# in fractional coordinates with respect to the lattice vectors
coordinate_frac = np.array(
    [
        [0.0, 0.0, 0.0],  # coordinates of basis site, b=0
    ]
).transpose()  # <--- note transpose


# Make atomic occupants with a discrete Cmagspin value (as an atom property)
def make_atom_with_Cmagspin(
    name: str,
    value: float,
):
    """Make an atomic occupant with a discrete collinear magnetic spin (Cmagspin) value."""
    return xtal.Occupant(
        name=name,  # "chemical name" of Occupant
        atoms=[
            xtal.AtomComponent(
                name=name,  # "chemical name" of the AtomComponent
                coordinate=np.array([0.0, 0.0, 0.0]),
                properties={
                    "Cmagspin": np.array(
                        [value]
                    )  # fixed properties of the AtomComponent
                },
            ),
        ],
    )


# Occupation degrees of freedom (DoF)
A_down_occ = make_atom_with_Cmagspin(name="A", value=-1.0)
A_up_occ = make_atom_with_Cmagspin(name="A", value=1.0)
occupants = {
    "A.down": A_down_occ,  # <label> : occupant
    "A.up": A_up_occ,  # <label> : occupant
}
occ_dof = [
    ["A.down", "A.up"],  # occupants allowed on basis site, b=0
]

# Construct the xtal.Prim
prim = xtal.Prim(
    lattice=lattice,
    coordinate_frac=coordinate_frac,
    occ_dof=occ_dof,
    occupants=occupants,
    title="Ising_occ",
)

Create a CASM project#

This step:

  • Initializes a Ising model CASM project using Project.init.

  • Prints the prim’s factor group symmetry operations using SymCommand.print_factor_group.

Important

Checking the factor group helps to catch errors in the prim. The factor group is used to determine which configurations and basis functions are symmetrically equivalent.

Notes

A CASM project is a directory containing data related to a particular prim. The CASM project directory structure standardizes the location of various files used by multiple CASM methods. This makes it easier to perform the most common operations and easier to share a project with others.

The Project class helps read and write and use CASM project data. A Project is initialized by specifying a prim and project directory location using Project.init. This will:

  1. Check if the prim has a primitive unit cell with a CASM standard lattice orientation

  2. Perform a symmetry analysis

  3. Generate some default directories, data, and settings

  4. Perform a configuration check

If a project already exists at the specified location, the prim argument to Project.init is ignored and the existing project is loaded.

Additional notes:

  • Project files that the user should not typically modify directly, including a copy of the prim, are stored in a hidden .casm sub-directory of the CASM project directory. The presense or absence of the .casm directory is used by CASM to detect a CASM project.

  • A description of symmetry analysis in CASM can be found in the libcasm-xtal usage documentation.

  • SymCommand.print_factor_group prints symmetry operations using notation which roughly follows the conventions of the International Tables for Crystallography.

from casm.project import Project
import pathlib

# Initialize the project
project = Project.init(
    path="Ising_occ",
    prim=prim,
)

# Coordinates can be printed in 'cart' or 'frac' coordinate mode
print("\nFactor group operations:")
project.sym.print_factor_group(coord="frac")

Enumerate symmetrically unique configurations#

This step:

  • Creates an enumeration with id “main” using EnumCommand.get().

  • Enumerates all symmetrically unique Configuration in 2d supercells of the first two lattice vectors, up to volume 10, using EnumData.occ_by_supercell().

  • Stores results in a ConfigurationSet and writes it to a file (config_set.json) in the enumeration directory.

Notes

A Configuration is a compact representation of the unit cell for a crystal structure that is consistent with the DoF specified in the prim. For this Ising model project, a configuration can be specified by:

  • the supercell of the prim that is the unit cell for the crystal structure, and

  • an integer occupation array (i.e. [1, 0, 1, …]), indicating if “A.up” or “A.down” is on each site in the supercell.

Enumeration directories are located in a CASM project at:

    enum_dir = <project>/enumerations/enum.<enum_id>/

The EnumData class helps interact with the saved Configuration. An EnumData object (enum) is created from the Project using EnumCommand.get() with the enumeration id:

    enum = project.enum.get(id="main")

The EnumData.occ_by_supercell() method:

  • enumerates configurations in a range of supercells specified by their volume with respect to the primitive unit cell volume,

  • inserts enumerated configurations in EnumData.configuration_set, and

  • writes EnumData.configuration_set to the config_set.json file in the enumeration directory.

CASM implements a strict ordering on configurations, and the configuration which compares greatest to all the other symmetrically equivalent configurations in a supercell is said to be in canonical form. By default, only primitive configurations that are in canonical form are stored by EnumData.occ_by_supercell(). Using this default, if configurations are enumerated from the volume 1 supercell on up, then only symmetrically unique configurations are stored.

# Enumerate configurations:
# - in supercells with volume 1 to 10
# - limit to 2d supercells (the "a" and "b" lattice vectors)
# - Configurations are stored in the enumeration's ConfigurationSet
enum = project.enum.get(id="main")
enum.occ_by_supercell(
    max=10,
    min=1,
    dirs="ab",
    verbose=True,
)

Iterate over enumerated configurations#

This step:

  • Iterates over the first 20 ConfigurationRecord in EnumData.configuration_set.

  • Prints the configuration’s name, occupation indices, atom types, and magnetic spin values.

Notes

For configuration stored in a ConfigurationSet, the configuration name consists of a supercell name and a configuration index. The supercell name (i.e. “SCEL2_2_1_1_0_1_1”) is generated from the transformation matrix that gives the supercell lattice vectors as a linear combination of the primitive cell lattice vectors. The configuration index is a sequential index generated by the ConfigurationSet for configurations in the same supercell.

When iterating over ConfigurationRecord in a EnumData.configuration_set, the ordering is based on sorting the configuration by supercell and then their DoF values, not their configuration name.

# Iterate over the first 20 ConfigurationRecord in the ConfigurationSet,
# and print:
# - the configuration name
# - the occupation indices
# - the corresponding atom types
# - the corresponding Cmagspin value
print(f'{"ConfigurationName":24}{"OccupationIndices":20}{"AtomType":36}{"Cmagspin":36}')
print("-" * 116)
for i, record in enumerate(enum.configuration_set):
    name = record.configuration_name
    occupation = record.configuration.occupation
    structure = record.configuration.to_structure()
    atom_type = structure.atom_type()
    Cmagspin = structure.atom_properties()["Cmagspin"][0, :]
    print(f"{name:24}{str(occupation):20}{str(atom_type):36}{str(Cmagspin):36}")
    if i == 20:
        break
