    "## Create a CASM project\n",
    "\n",
    "**This step:**\n",
    "\n",
    "- Initializes a diamond cubic Si-Ge alloy CASM project using [Project.init].\n",
    "- Prints the prim’s factor group symmetry operations using [SymCommand.print_factor_group].\n",
    "\n",
    ":::{important}\n",
    "Checking the factor group helps to catch errors in the prim. The factor group is used to determine which configurations and basis functions are symmetrically equivalent.\n",
    ":::\n",
    "\n",
    ":::{admonition} **Notes**\n",
    ":class: dropdown note\n",
    "\n",
    "A CASM project is a directory containing data related to a particular prim. The CASM project directory structure standardizes the location of various files used by multiple CASM methods. This makes it easier to perform the most common operations and easier to share a project with others.\n",
    "\n",
    "The [Project] class helps read and write and use CASM project data. A [Project] is initialized by specifying a prim and project directory location using [Project.init]. This will:\n",
    "\n",
    "1. Check if the prim has a primitive unit cell with a CASM standard lattice orientation\n",
    "2. Perform a symmetry analysis\n",
    "3. Generate some default directories, data, and settings\n",
    "4. Perform a configuration check\n",
    "\n",
    "If a project already exists at the specified location, the *prim* argument to [Project.init] is ignored and the existing project is loaded.\n",
    "\n",
    "**Additional notes**:\n",
    "\n",
    "- Project files that the user should not typically modify directly, including a copy of the prim, are stored in a hidden *.casm* sub-directory of the CASM project directory. The presense or absence of the *.casm* directory is used by CASM to detect a CASM project.\n",
    "- A description of symmetry analysis in CASM can be found in the [libcasm-xtal usage documentation].\n",

    "## Enumerate symmetrically unique configurations\n",
    "\n",
    "**This step:**\n",
    "\n",
    "- Creates an enumeration with id \"main\" using [EnumCommand.get()].\n",
    "- Enumerates all symmetrically unique [Configuration] in supercells up to volume 4 using [EnumData.occ_by_supercell()].\n",
    "- Stores results in a [ConfigurationSet] and writes it to a file (*config_set.json*) in the enumeration directory.\n",
    "\n",
    ":::{admonition} **Notes**\n",
    ":class: dropdown note\n",
    "\n",
    "A [Configuration] is a compact representation of the unit cell for a crystal structure that is consistent with the DoF specified in the prim. For this Si-Ge project, a configuration can be specified by:\n",
    " \n",
    "  - the supercell of the prim that is the unit cell for the crystal structure, and\n",
    "  - an integer occupation array (i.e. [1, 0, 1, ...]), indicating if Si or Ge is on each site in the supercell.\n",
    "\n",
    "Enumeration directories are located in a CASM project at:\n",
    "\n",
    "```\n",
    "    enum_dir = <project>/enumerations/enum.<enum_id>/\n",
    "```\n",
    "\n",
    "The [EnumData] class helps interact with the saved [Configuration]. An [EnumData] object (*enum*) is created from the [Project] using [EnumCommand.get()] with the enumeration id:\n",
    "\n",
    "```python\n",
    "    enum = project.enum.get(id=\"main\")\n",
    "```\n",
    "\n",
    "The [EnumData.occ_by_supercell()] method:\n",
    "\n",
    "- enumerates configurations in a range of supercells specified by their volume with respect to the primitive unit cell volume,\n",
    "- inserts enumerated configurations in [EnumData.configuration_set], and\n",
    "- writes [EnumData.configuration_set] to the *config_set.json* file in the enumeration directory.\n",
    "\n",
    "CASM implements a strict ordering on configurations, and the configuration which compares greatest to all the other symmetrically equivalent configurations in a supercell is said to be in canonical form. By default, only primitive configurations that are in canonical form are stored by [EnumData.occ_by_supercell()]. Using this default, if configurations are enumerated from the volume 1 supercell on up, then only symmetrically unique configurations are stored.\n",

    "## Set parametric composition axes\n",
    "\n",
    "**This step**:\n",
    "\n",
    "- Prints the standard choices for independent parametric composition axes using [Project.chemical_composition_axes].\n",
    "- Selects one choice to use with [CompositionAxes.set_current_axes()]\n",
    "- Prints the selected axes along with formulas for converting between (i) number composition and parametric composition and (ii) chemical potential and parametric chemical potential.\n",
    "- Saves the chosen parametric composition axes with [CompositionAxes.commit()].\n",
    "\n",
    ":::{admonition} **Notes**\n",
    ":class: dropdown note\n",
    "\n",
    "In a crystal with a fixed number of sites, the number of species occupying the same sublattice are not indepedent. In general, a crystal occupied by $s$ component species will have $k<s$ independent compositions. As described in [\\[1\\]](#ref-1), CASM converts between compositions expressed as number of species per unit cell and compositions in terms of independent \"parametric composition axes\" using:\n",
    "\n",
    "\\begin{align}\n",
    "    \\vec{n} &= \\vec{n}_0 + \\mathbf{Q} \\vec{x} \\\\\n",
    "    \\vec{x} &= \\mathbf{R}^{\\mathsf{T}} (\\vec{n} - \\vec{n}_0)\n",
    "\\end{align}\n",
    "\n",
    "\n",
    "where:\n",
    "\n",
    "- $\\vec{n}$: Vector of shape=($s$,), the number of each component species\n",
    "  per unit cell (*mol_composition*).\n",
    "- $\\vec{x}$: Vector of shape=($k$,), The composition along each composition axis when referenced to the origin composition (*param_composition*).\n",
    "- $\\vec{n}_0$: Vector of shape=($s$,), The origin in composition space, as the number of each component species per unit cell.\n",
    "- $\\mathbf{Q}$: Matrix of shape=($s$, $k$), with columns representing\n",
    "the change in composition per unit cell going one unit distance along each independent composition axis.\n",
    "- $\\mathbf{R}$: Matrix of shape=($s$, $k$), such that $\\mathbf{R}^{\\mathsf{T}}\\mathbf{Q} = \\mathbf{Q}^{\\mathsf{T}}\\mathbf{R} = \\mathbf{I}$.\n",
    "\n",
    "The parametric composition axes are the columns of $\\mathbf{Q}$, $\\vec{q}_i$. Due to preservation of the number of sites per unit cell, $\\sum_{i} Q_{ij} = 0$. If vacancies are allowed, they are included as a component species.\n",
    "\n",
    "(ref-1)=\n",
    "\\[1\\]: *Computational Materials Science* 260 (2025) 114091. DOI: [10.1016/j.commatsci.2025.114091](https://doi.org/10.1016/j.commatsci.2025.114091).\n",
    "\n",
    ":::\n",
    "\n",
    "[Project.chemical_composition_axes]: https://prisms-center.github.io/CASMcode_pydocs/casm/project/2.0/reference/casm/_autosummary/casm.project.Project.chemical_composition_axes.html#chemical-composition-axes\n",

    "\n",
    "**This step**:\n",
    "\n",
    "- Constructs a [ConfigSelection] using [EnumData.config_selection()].\n",

    ":::{admonition} **Notes**\n",
    ":class: dropdown note\n",
    "\n",
    "[ConfigSelection] is a helper that can be used to:\n",
    "\n",
    "- set certain configurations in an enumeration as \"selected\" and others as \"unselected\",\n",
    "- iterate over only selected configurations (by default), or all configurations, or only unselected configurations,\n",
    "- evaluate configuration properties, such as the parametric composition or cluster expansion basis function correlations, using data stored in the project, and\n",
    "- perform actions on selected configurations, like setup calculation input files or collect calculation results.\n",
    "\n",
    "[ConfigSelection] are given a *name* and stored in a file named *config_selection.\\<name\\>.json* in the enumeration directory when the [ConfigSelection.commit()] method is used. By default, when a [ConfigSelection] does not already exist it is constructed with all configurations selected.\n",
    "\n",
    "When [ConfigSelection] is constructed or when a certain property of a configuration is requested from a [ConfigSelectionRecord], project data like the chemical composition axes or a CASM clexulator is read and used to evaluate the requested property.\n",
    "\n",
    "\n",
    "**Additional notes**:\n",
    "- Enumerations also allow storing configurations in a list, [EnumData.configuration_list], which is stored in a file named *config_list.json*, to enable storing configurations that are not in the canonical supercell (for example, configurations with a particular supercell orientation).\n",

    "## Write structure files\n",
    "\n",
    "**This step**:\n",
    "\n",
    "- Iterates over selected configurations using [ConfigSelection].\n",
    "- In the calculation directory for each configuration, writes:\n",
    "  - the CASM [Configuration] to a file named *config.json*,\n",
    "  - a CASM [Structure] representation to a file named *structure.json*, and\n",
    "  - a VASP [POSCAR] to a file named *structure.vasp*.\n",
    "\n",
    "\n",
    ":::{admonition} **Notes**\n",
    ":class: dropdown note\n",
    "\n",
    "A CASM [Structure]:\n",
    "\n",
    "- represents a crystal structure with a 3d lattice,\n",
    "- is not restricted to the DoF values allowed by a prim,\n",
    "- has built in methods for conversions to and from VASP POSCAR format.\n",
    "\n",
    "The [ConfigSelectionRecord.structure] property does the conversion from [Configuration] to [Structure], leaving out any vacancies, using the [Configuration.to_structure] method. It is also possible get the [Structure] representation including vacancies using the [ConfigSelectionRecord.structure_with_vacancies] property.\n",
    "\n",
    "Files are written to the CASM standard calculation directory for the configuration, which is:\n",
    "\n",
    "    calc_dir = <enum_dir>/training_data/calctype.<calctype_id>/<configname>/\n",
    "\n",
    "where the default *calctype_id* is \"default\".\n",
    ":::\n",
