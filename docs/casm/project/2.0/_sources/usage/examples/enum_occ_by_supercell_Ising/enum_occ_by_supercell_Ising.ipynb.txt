    "## Specify the \"prim\"\n",
    "\n",
    "**This step:**\n",
    "\n",
    "- Defines the primitive crystal structure and allowed degrees of freedom (DoF) for a 2-dimensional Ising model system.\n",
    "- Uses occupants named \"A.down\" and \"A.up\" to represent \"A\" atoms with collinear magnetic spin values of -1 and +1, respectively.\n",
    "\n",
    ":::{admonition} **Notes**\n",
    ":class: dropdown note\n",
    "\n",
    "A primitive crystal structure and allowed degrees of freedom (the \"prim\") specifies:\n",
    "\n",
    "- primitive cell lattice vectors\n",
    "- crystal basis sites\n",
    "- global DoF (none in this case)\n",
    "- site DoF (A.up or A.down occupation in this case).\n",
    "\n",
    "Here the prim is constructed using the [xtal.Prim]. \n",
    "\n",
    "- [xtal.Prim]: This is just the prim definition\n",
    "- [casmconfig.Prim]: This is the [xtal.Prim], along with the symmetry representations needed for applying symmetry operations to CASM [Configuration]\n",
    ":::\n",
    "\n",
    "[xtal.Prim]: https://prisms-center.github.io/CASMcode_pydocs/libcasm/xtal/2.0/reference/libcasm/_autosummary/libcasm.xtal.Prim.html#prim\n",
    "[casmconfig.Prim]: https://prisms-center.github.io/CASMcode_pydocs/libcasm/configuration/2.0/reference/libcasm/_autosummary/libcasm.configuration.Prim.html#prim\n",
    "[Configuration]: https://prisms-center.github.io/CASMcode_pydocs/libcasm/configuration/2.0/reference/libcasm/_autosummary/libcasm.configuration.Configuration.html#configuration"
   ]

    "import libcasm.xtal as xtal\n",
    "import numpy as np\n",
    "\n",
    "# Lattice vectors:\n",
    "lattice = xtal.Lattice(\n",
    "    np.array(\n",
    "        [\n",
    "            [1.0, 0.0, 0.0],  # 1st lattice vector\n",
    "            [0.0, 1.0, 0.0],  # 2nd lattice vector\n",
    "            [0.0, 0.0, 10.0],  # 3rd lattice vector (will have no effect)\n",
    "        ]\n",
    "    ).transpose()  # <--- note transpose\n",
    ")\n",
    "\n",
    "# Basis sites positions, as columns of a matrix,\n",
    "# in fractional coordinates with respect to the lattice vectors\n",
    "coordinate_frac = np.array(\n",
    "    [\n",
    "        [0.0, 0.0, 0.0],  # coordinates of basis site, b=0\n",
    "    ]\n",
    ").transpose()  # <--- note transpose\n",
    "\n",
    "\n",
    "# Make atomic occupants with a discrete Cmagspin value (as an atom property)\n",
    "def make_atom_with_Cmagspin(\n",
    "    name: str,\n",
    "    value: float,\n",
    "):\n",
    "    \"\"\"Make an atomic occupant with a discrete collinear magnetic spin (Cmagspin) value.\"\"\"\n",
    "    return xtal.Occupant(\n",
    "        name=name,  # \"chemical name\" of Occupant\n",
    "        atoms=[\n",
    "            xtal.AtomComponent(\n",
    "                name=name,  # \"chemical name\" of the AtomComponent\n",
    "                coordinate=np.array([0.0, 0.0, 0.0]),\n",
    "                properties={\n",
    "                    \"Cmagspin\": np.array(\n",
    "                        [value]\n",
    "                    )  # fixed properties of the AtomComponent\n",
    "                },\n",
    "            ),\n",
    "        ],\n",
    "    )\n",
    "\n",
    "\n",
    "# Occupation degrees of freedom (DoF)\n",
    "A_down_occ = make_atom_with_Cmagspin(name=\"A\", value=-1.0)\n",
    "A_up_occ = make_atom_with_Cmagspin(name=\"A\", value=1.0)\n",
    "occupants = {\n",
    "    \"A.down\": A_down_occ,  # <label> : occupant\n",
    "    \"A.up\": A_up_occ,  # <label> : occupant\n",
    "}\n",
    "occ_dof = [\n",
    "    [\"A.down\", \"A.up\"],  # occupants allowed on basis site, b=0\n",
    "]\n",
    "\n",
    "# Construct the xtal.Prim\n",
    "prim = xtal.Prim(\n",
    "    lattice=lattice,\n",
    "    coordinate_frac=coordinate_frac,\n",
    "    occ_dof=occ_dof,\n",
    "    occupants=occupants,\n",
    "    title=\"Ising_occ\",\n",
    ")"
   ]

    "## Create a CASM project\n",
    "\n",
    "**This step:**\n",
    "\n",
    "- Initializes a Ising model CASM project using [Project.init].\n",
    "- Prints the prim’s factor group symmetry operations using [SymCommand.print_factor_group].\n",
    "\n",
    ":::{important}\n",
    "Checking the factor group helps to catch errors in the prim. The factor group is used to determine which configurations and basis functions are symmetrically equivalent.\n",
    ":::\n",
    "\n",
    ":::{admonition} **Notes**\n",
    ":class: dropdown note\n",
    "\n",
    "A CASM project is a directory containing data related to a particular prim. The CASM project directory structure standardizes the location of various files used by multiple CASM methods. This makes it easier to perform the most common operations and easier to share a project with others.\n",
    "\n",
    "The [Project] class helps read and write and use CASM project data. A [Project] is initialized by specifying a prim and project directory location using [Project.init]. This will:\n",
    "\n",
    "1. Check if the prim has a primitive unit cell with a CASM standard lattice orientation\n",
    "2. Perform a symmetry analysis\n",
    "3. Generate some default directories, data, and settings\n",
    "4. Perform a configuration check\n",
    "\n",
    "If a project already exists at the specified location, the *prim* argument to [Project.init] is ignored and the existing project is loaded.\n",
    "\n",
    "**Additional notes**:\n",
    "\n",
    "- Project files that the user should not typically modify directly, including a copy of the prim, are stored in a hidden *.casm* sub-directory of the CASM project directory. The presense or absence of the *.casm* directory is used by CASM to detect a CASM project.\n",
    "- A description of symmetry analysis in CASM can be found in the [libcasm-xtal usage documentation].\n",
    "- [SymCommand.print_factor_group] prints symmetry operations using notation which roughly follows the conventions of the [International Tables for Crystallography].\n",

    "## Enumerate symmetrically unique configurations\n",
    "\n",
    "**This step:**\n",
    "\n",
    "- Creates an enumeration with id \"main\" using [EnumCommand.get()].\n",
    "- Enumerates all symmetrically unique [Configuration] in 2d supercells of the first two lattice vectors, up to volume 10, using [EnumData.occ_by_supercell()].\n",
    "- Stores results in a [ConfigurationSet] and writes it to a file (*config_set.json*) in the enumeration directory.\n",
    "\n",
    ":::{admonition} **Notes**\n",
    ":class: dropdown note\n",
    "\n",
    "A [Configuration] is a compact representation of the unit cell for a crystal structure that is consistent with the DoF specified in the prim. For this Ising model project, a configuration can be specified by:\n",
    " \n",
    "  - the supercell of the prim that is the unit cell for the crystal structure, and\n",
    "  - an integer occupation array (i.e. [1, 0, 1, ...]), indicating if \"A.up\" or \"A.down\" is on each site in the supercell.\n",
    "\n",
    "Enumeration directories are located in a CASM project at:\n",
    "\n",
    "```\n",
    "    enum_dir = <project>/enumerations/enum.<enum_id>/\n",
    "```\n",
    "\n",
    "The [EnumData] class helps interact with the saved [Configuration]. An [EnumData] object (*enum*) is created from the [Project] using [EnumCommand.get()] with the enumeration id:\n",
    "\n",
    "```python\n",
    "    enum = project.enum.get(id=\"main\")\n",
    "```\n",
    "\n",
    "The [EnumData.occ_by_supercell()] method:\n",
    "\n",
    "- enumerates configurations in a range of supercells specified by their volume with respect to the primitive unit cell volume,\n",
    "- inserts enumerated configurations in [EnumData.configuration_set], and\n",
    "- writes [EnumData.configuration_set] to the *config_set.json* file in the enumeration directory.\n",
    "\n",
    "CASM implements a strict ordering on configurations, and the configuration which compares greatest to all the other symmetrically equivalent configurations in a supercell is said to be in canonical form. By default, only primitive configurations that are in canonical form are stored by [EnumData.occ_by_supercell()]. Using this default, if configurations are enumerated from the volume 1 supercell on up, then only symmetrically unique configurations are stored.\n",
    ":::\n",
    "\n",
    "[EnumCommand.get()]: https://prisms-center.github.io/CASMcode_pydocs/casm/project/2.0/reference/casm/_autosummary/casm.project.enum.EnumCommand.get.html#get\n",

    "## Iterate over enumerated configurations\n",
    "\n",
    "**This step:**\n",
    "\n",
    "- Iterates over the first 20 [ConfigurationRecord] in [EnumData.configuration_set].\n",
    "- Prints the configuration's name, occupation indices, atom types, and magnetic spin values.\n",
    "\n",
    ":::{admonition} **Notes**\n",
    ":class: dropdown note\n",
    "\n",
    "For configuration stored in a [ConfigurationSet], the configuration name consists of a supercell name and a configuration index. The supercell name (i.e. \"SCEL2_2_1_1_0_1_1\") is generated from the transformation matrix that gives the supercell lattice vectors as a linear combination of the primitive cell lattice vectors. The configuration index is a sequential index generated by the [ConfigurationSet] for configurations in the same supercell.\n",
    "\n",
    "When iterating over [ConfigurationRecord] in a [EnumData.configuration_set], the ordering is based on sorting the configuration by supercell and then their DoF values, not their configuration name.\n",
